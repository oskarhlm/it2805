Questions

  1. The server is the central unit in the server-client architecture, which in its simplest form is a one-to-many relationship. A server will typically answer requests from a client and serves as the provider of resources or services. A client will typically request that the server returns a webpage stored on the server. The server will then return a bundle of HTML, CSS and Javascript for the client (the web browser) to render.

  2. The client initiates communication between itself and the server. It requests of the server things like:

    • Return me a webpage that I can render on-screen

    • Save this form data in a database

    • Get a list of users

    After a request is sent, the client awaits the resonse and the handles it in some specified way.

  3. An IP adress is a unique identifier that every device or network on the internet is required to have. For long we have used IPv4 to represent IP adresses as 32-bit numbers, but recently we have had to adopt IPv6 (128-bit) to account for the growth of the internet.

  4. The DNS (Domain Name System) is a naming system that translates domain names to numerical IP adresses.

  5. The internet is a globally connected network system which is used for communication and data retrieval across the globe. The WWW is a collection of information which is access via the internet. It is built on top of the internet, and is sort of the software if we think of the internet as the hardware.

  6. HyperText Markup Language (HTML) is a language used to design documents/pages to be displayed in a web browser. It defines the semantics of the webpage, using open and closing tags. Many tags have semantic meaning (not all, see div, span, etc.) that helps us navigate it.

  7. CSS is not document content and thus it should be in the head, not in the body. Also, when you include the CSS files inside the head (before the body), your styles will load before the rest of the document. This may be undesirable since your page can (will) look very ugly if it has no CSS.

  8. Inline CSS has higher specificity that the rules defined in style, that is, they rules defined in style will be overridden.
